Introducing Active CPU pricing for Fluid compute

Fluid compute now uses Active CPU pricing. Only pay CPU rates when your function is actively computing. Building on existing Fluid gains, this brings additional savings of up to 90% for workloads like LLM calls, AI agents, or tasks with idle time.

Key Benefits

  • Only pay for active CPU computation time
  • Up to 90% savings for idle workloads
  • Perfect for LLM calls and AI agents
  • Seamless integration with existing Fluid compute

Getting Started

Active CPU pricing is automatically enabled for all new Fluid compute functions. Existing functions will be migrated gradually over the coming weeks.
